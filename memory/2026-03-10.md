# 2026-03-10 — Esports Latency Arb Bot (Rebuilt from Spread-Arb)

## Strategy
Completely replaced the spread-arb bot with an esports latency arbitrage strategy.
Core idea: we receive real-time game data BEFORE the Twitch broadcast catches up.
Buy positions on the winning team immediately after pivotal events, sell 28s later when broadcast viewers reprice the market.

## Four Games Supported

| Game | Data Source | Key Triggers | Broadcast Edge |
|---|---|---|---|
| LoL | `feed.lolesports.com` (Riot, 2s poll) | Kill cluster, Baron, Dragon, Inhibitor, Gold swing | ~20-25s |
| Valorant VCT | Same Riot feed (auto via getLive) | Round win, Spike plant/defuse, Kill cluster, Ace | ~20-25s |
| CS2 | HLTV scorebot WebSocket (real-time push) | Round win, Bomb plant, Ace | ~60-90s |
| Dota 2 | OpenDota `/api/live` (no key needed) | Kill cluster, Gold swing, Tower fall, Barracks fall | ~2.5 MINUTES |

## Files
- `/root/polymarket-bot/src/server.ts` — main bot (esports strategy)
- `/root/polymarket-bot/src/trader.ts` — Polymarket CLOB client (unchanged)
- `/root/polymarket-bot/src/cs2.ts` — CS2/HLTV module
- `/root/polymarket-bot/src/valorant.ts` — VCT module
- `/root/polymarket-bot/src/dota2.ts` — Dota 2/OpenDota module
- `/root/polymarket-bot/src/server.ts.spread-arb.bak` — old strategy backup

## Config (.env)
- `DRY_RUN=true` — paper trading until manually set to false
- `TRADE_SIZE_USDC=10` — $10 per position
- `HOLD_SECONDS=28` — exit after 28s
- `MIN_KILL_CLUSTER=3` — 3+ kills in 12s = trigger
- `GOLD_SWING_TRIGGER=3000` — gold swing threshold
- `POLYMARKET_KEYWORDS=league,lol,...,dota,dota2,international,dpc`

## Start/Stop
```bash
# Start
cd /root/polymarket-bot && nohup npm run dashboard > /tmp/polymarket-bot.log 2>&1 &
# Stop all
ps aux | grep "tsx src/server.ts" | grep -v grep | awk '{print $2}' | xargs kill -9
# Dashboard
http://178.156.248.17:8080
# Logs
tail -f /tmp/polymarket-bot.log
```

## To go live
Change `DRY_RUN=false` in `/root/polymarket-bot/.env` and restart.

## Key insight
Dota 2 has the biggest edge — OpenDota queries Steam GC directly (no delay), while Twitch has the 2-minute built-in DotaTV delay + encoding delay = ~2.5 minute window to trade before the market reprices.
